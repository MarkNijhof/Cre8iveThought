Ik wil mij zelf graag even introduceren; mijn naam is Mark Nijhof en ik ben momenteel woonachtig in Oslo Noorwegen, maar ik ben Nederlandse. Ik ben getrouwd met een Noorse en heb inmiddels 3 kinderen. Momenteel wij zijn nu aan het plannen om terug te keren naar Nederland; eind April om precies the zijn. Linda Lotte introduceerde mij met de vacature voor een Senior Front-end / JavaScript ontwikkelaar en van wat zij mij heeft verteld over uw bedrijf en de baan ben ik ontzettend enthousiast om verder te kijken of wij goed bij elkaar zouden passen.

Ik ben een zeer gedreven en leergierige ontwikkelaar en heb tijdens mijn loopbaan gewerkt met C# .Net, Deplhi, Perl, PHP maar ben de laatste 8 maanden eigenlijk uitsluitend met Ruby aan het werk geweest. Het merendeel van de projecten waar ik aan gewerkt heb zijn web gebaseerd en hand gemaakt, hierdoor heeft JavaScript, HTML en CSS altijd een heel belangrijke rol voor mij gehad. Dit is ook iets dat ik ontzettend leuk vind om mee te werken. Ruby heeft inmiddels ook mijn hart veroverd. Bij mijn huidige baan maar ook mijn priv√© projecten zijn nu allemaal in Ruby. Hier is een lijst met frameworks waar ik momenteel mee werk: Ruby, JavaScript, HTML, CSS, Rails3, Sinatra, Padrino, Haml, SASS, Compass, MongoDB, CouchDB, RSpec, Cucumber, JSpec, Watchr en vele andere Gems die de ontwikkeling helpen. Dit samen met Heroku is werkelijk een hele krachtige oplossing. Ook heb ik Windows nu ook al 8 maanden niet gezien :)

Ik ben een groot voorstander van testen, meer specifiek Test-Driven Development, en niet alleen de Server code, maar ook de JavaScript code die ik schrijf. Ik heb eerder voornamelijk met JSpec gewerkt voor het testen van mijn JavaScript code, maar helaas is de ontwikkeling daarvan gestopt en ga ik waarschijnlijk verder met Jasmine. Een van de dingen die mij erg aanspraken van JSpec was dat het niet erg op JavaScript code leek. Dus wanneer het teveel op JavaScript begint te lijken weet ik dat mijn tests te complex zijn. Daarom wil ik ook bekijken of ik CoffeeScript kan gebruiken in combinatie met Jasmine. Daarnaast genereert CoffeeScript ook altijd correcte JavaScript code wat weer minder cross-browser problemen geeft. Voor de server code gebruik ik RSpec en Cucumber.

Ik heb over Test-Driven JavaScript gepresenteerd bij NDC 2010 (Norwegian Developer Conference) en Smidig 2010 (Agile conference in Oslo), beide sessies zijn opgenomen en de links zijn hier onderaan toegevoegd.

Verder ben ik ook actief in de community, ik zit in de board van de Oslo XP, Node.js Oslo en NNUG Oslo gebruikers groepen, waar wij meetings organiseren en sprekers uitnodigen. B.v. op 24 januari hebben we een gezamenlijke meeting met XP en Node.js waar we Christian Johansen (schrijver van Test-Driven JavaScript Development) en Fabian Jakobs (van de Cloud9 IDE) hebben uitgenodigd om voor onze gebruikers te presenteren over Test-Driven JavaScript. Ook help ik nu met het opzetten van ROOTS 2011, voornamelijk probeer ik speakers te krijgen voor de conferentie. Ook ben ik in persoonlijk contact met verschillende belangrijke personen uit de software wereld en hierdoor heb ik al verschillende kunnen uitnodigen bij de bedrijven waarvoor ik heb gewerkt.

Ik heb ook van Linda Lotte begrepen dat het mogenlijk is om remote te werken zodat ik niet elke dag in het verkeer naar Amsterdam hoef te zitten. En dat is erg gewaardeerd, graag wil ik u ook vertellen dat het huis dat wij gaan huren een aparte studio heeft dat ik zal gebruiken als thuis kantoor. Een ideale plek om goed geconsentreerd te kunnen werken. Verder heb ik eerder ook succesvol veel thuis gewerkt.

Vanaf februari heb ik 10 weken ouderschaps verlof en ik zou tijdens dit verlof ook al kunnen beginnen met werk, dit zal echter wel in Noorwegen zijn en niet full-time.
