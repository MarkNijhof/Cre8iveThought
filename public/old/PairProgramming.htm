Pair Programming

Pair Programming is a software development practice that has its origin from Extreme Programming (XP). The idea was that since code reviews are a good way to unsure good code quality, the XP crowd took that idea to the extreme by making code reviews a continuous process.

How it works

When Pair Programming, a pair of two developers (or for example a developer and a tester) will work on the same machine trying to solve a problem. One person will be the driver and have the keyboard to do the actual typing. The other person will be the navigator who’s responsibility it is to think about design issues and other potential problems.

The developers constantly switch between the roles of driver and navigator. There are many different guidelines on how often this switching should happen. We noticed that when the pair gets more comfortable together it will come very natural when to switch, however in the beginning it may be helpful to set some strict rules on when to switch.

Pairing is also a great way of teaching or introducing something to another person, we noticed in this situation that the person teaching usually takes the role of navigator.

Benefits

  • Higher overall code quality and design
  • Faster development of new features
  • Better sharing of knowledge within the team
  • Improved moral

At Cre8ive Thought we prefer to Pair Program especially on the more complex tasks or when teaching new technologies. Pair Programming and Test-Driven Development really is a perfect match.
